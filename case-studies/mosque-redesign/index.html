Redesigning Bilal Masjid's About Page: Improving Usability, Accessibility, and Responsiveness

Context / Problem

The current Bilal Masjid website showing dated design and usability issues.

For as long as I can remember, the Bilal Masjid website has looked dated and lacked modern usability features. Navigation is cluttered, visual hierarchy is weak, and the site isn't responsive—making it frustrating to use, especially on mobile devices. These issues present barriers for users trying to learn more about the mosque or engage with its programs.

I focused on the "About Us" section, a key entry point for new visitors. Despite having minimal content, the section is split into multiple pages with inconsistent structure and labels. Prayer times are relegated to the sidebar, buttons don't appear clickable, and the site suffers from poor accessibility, including low color contrast and missing alt text. My goal was to modernize this page to make it clean, responsive, and accessible while serving as a model for future redesigns across the site.

Audience

The intended users of the Bilal Masjid website include a wide range of visitors: new community members looking to learn about the mosque, regular attendees checking prayer times or events, and donors seeking quick access to support options. These users vary in age, tech experience, and access needs. Some visit the site from desktops; others use mobile devices. A successful redesign would need to meet the expectations of both first-time and returning users: efficient, easy to navigate, and inclusive for all.

Redesign Process

The redesign aimed to create a clean, accessible, and mobile-friendly webpage by merging the History and About Us subsections into a single unified About Us page. To visualize the redesign and ensure consistency, I created detailed mockups and a comprehensive design style guide in Figma. The guide included typography standards, button styles, color palettes, and responsive layouts. Mockups were designed for mobile, tablet, and desktop views to validate the responsive design before implementation.

Key Improvements

Streamlined Navigation

  • Simplified content by combining related sections (About Us + History)
  • Removed redundant sidebar elements
  • Implemented a mobile-friendly hamburger menu

Responsive & Mobile-First Design

  • Designed layouts for all devices using Flexbox, Grid, and media queries
  • Improved font sizing and tap targets for mobile users
  • Components collapse and realign fluidly across breakpoints

Improved Visual and Button Design

  • Clear, interactive buttons with hover states
  • Strong visual hierarchy with consistent headings and CTAs
  • Mobile layout places key actions near the search bar for ease of access

Predictable UI Patterns

  • Adopted standard UX conventions like dropdown menus and consistent labeling
  • Removed duplicate or confusing labels (e.g., unified "Zakat" and "Donations")

Enhanced Interactivity

  • Introduced dynamic year-selector buttons for historical content
  • Improved engagement with interactive but unobtrusive components

Footer Optimization

  • Added a persistent footer with key links like "Subscribe," reducing the need to navigate away just to join the mailing list

Accessibility Improvements

To ensure full WCAG compliance, I made the following changes:

  • Alt Text: Added for all informative images
  • Color Contrast: Improved using dark-on-light combinations
  • ARIA Labels: Applied to buttons, forms, and navigation elements
  • WAVE Report: Conducted testing to confirm improved structure, hierarchy, and screen reader compatibility

Reflection

This project deepened my understanding of responsive design and accessibility. Technically, it gave me hands-on experience with CSS Flexbox, Grid, and media queries. I also used tools like clamp() to manage responsive typography.

More importantly, I learned how subtle choices (e.g., whether a button looks clickable, or whether content adapts well to smaller screens) can determine whether a website feels intuitive or frustrating. By centering user needs and accessibility in the design process, even small improvements can make a big difference in how inclusive and effective a website becomes.
